# React学習プロジェクト

プログラミング入門レベルからReactでwebアプリを作れるようになるための学習プロジェクトです。

## 📁 プロジェクト構成

```
learning-journey-frontend/
├── todo-app/              # ToDoアプリ（Phase 1完了）
│   ├── src/
│   ├── package.json
│   └── vite.config.js
├── timer-app/             # カウントダウンタイマー（Phase 2-1完了）
│   ├── src/
│   ├── package.json
│   └── vite.config.js
├── free-practice-app/     # 自由練習用アプリ（学習スケジュール外）
│   ├── src/
│   ├── package.json
│   └── vite.config.js
├── assignments/           # お題の仕様書
├── learning-journey/      # 学習記録
└── README.md
```

各アプリは完全に独立したプロジェクトです。

**free-practice-app について:**
- 学習スケジュールとは別に、自由に実験・練習するためのアプリ
- 写経なしで、自分でお題を考え、イメージしたものを少しずつコードにしていく
- React Routerを使ったランディングページなど、自由な実践の場

### 起動方法

```bash
# ToDoアプリを起動
cd todo-app
npm install  # 初回のみ
npm run dev

# Timerアプリを起動
cd timer-app
npm install  # 初回のみ
npm run dev

# 自由練習用アプリを起動
cd free-practice-app
npm install  # 初回のみ
npm run dev
```

## 🧪 ToDoアプリ - テスト

ToDoアプリのディレクトリで実行：

### Unit Test実行

```bash
cd todo-app
npm run test
```

**テストケース:**
- タイトルが表示される
- ToDoを追加できる
- 空のToDoは追加できない
- ToDoを削除できる
- 複数のToDoを追加できる

### Storybook

```bash
cd todo-app
npm run storybook
```

ブラウザで `http://localhost:6006` を開くと、以下のストーリーが確認できます：

- **Empty**: 初期状態（空のリスト）
- **WithOneItem**: ToDoが1つある状態
- **WithMultipleItems**: 複数のToDoがある状態
- **WithLongText**: 長いテキストのToDo

## 🎯 学習計画とロードマップ

### 学習の進め方

#### 基本的な学習スタイル：写経（サンプルコードを書き写す）
1. サンプルコードを提示してもらう
2. 自分でコードを書き写す
3. 動作確認
4. **コードの全ての部分を理解するまで質問を繰り返す**
5. 学習記録を`learning-journey/`に追記
6. Gitにコミット・push

**すべてのフェーズで写経スタイルを採用** - お題に対してサンプルコードを提示してもらい、理解しながら実装していく

#### free-practice-appでの自由実践
- 学習スケジュールとは完全に独立
- 自分でお題を考え、自分で実装方針を決める
- 必要に応じてサポートを受けながら、自由に実験・チャレンジする場
