### 2. 計算機アプリの状態設計
- **計算機に必要な状態の理解**:
  - `display`: 画面に表示する値（文字列）
  - `firstValue`: 最初に入力した値を保存（演算に使用）
  - `operator`: 選択された演算子（+, -, *, /）を保存
  - `shouldResetDisplay`: 次の入力時にディスプレイをリセットすべきかのフラグ
- **計算機の動作フロー**:
  1. 数字ボタンを押す → displayに表示
  2. 演算子ボタンを押す → firstValueに現在の値を保存、operatorに演算子を保存
  3. 次の数字を入力 → shouldResetDisplayフラグに基づいてdisplayを更新
  4. =ボタンを押す → firstValueと現在のdisplayの値で計算実行

### 計算機の状態管理設計

```jsx
const [display, setDisplay] = useState('0')           // 画面表示
const [firstValue, setFirstValue] = useState(null)     // 最初の値
const [operator, setOperator] = useState(null)         // 演算子
const [shouldResetDisplay, setShouldResetDisplay] = useState(false) // リセットフラグ
```

この4つの状態で計算機の動作を制御：
- `display`: ユーザーが見る画面（初期値'0'）
- `firstValue`: 演算子を押した時の値を保存（初期値null）
- `operator`: どの演算を実行するか（初期値null）
- `shouldResetDisplay`: 演算子を押した後、次の数字入力時にdisplayをリセットするか（初期値false）
