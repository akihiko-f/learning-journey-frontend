## 学習メモ

### プロジェクト完了時のチェックリスト
プロジェクトを完了する際には以下を必ず実施:
1. README.mdの「実装した機能」セクションを更新
2. 学習計画セクションで該当項目を完了マーク（✅）
3. 完了日を記録
4. 関連する学習記録へのリンクを追加
5. 必要に応じてプロジェクト構成図を更新

### Viteでの新規プロジェクト作成コマンド
```bash
# Node.jsがインストールされている前提
npm create vite@latest favorites-app -- --template react

# プロジェクトディレクトリに移動
cd favorites-app

# 依存関係をインストール
npm install

# 開発サーバーを起動
npm run dev
```

### モノレポ構成のメリット
今回のプロジェクト構成では、複数のReactアプリを1つのリポジトリで管理:
- 各プロジェクトは独立して動作
- 学習の進捗を一覧で把握しやすい
- 共通のドキュメント（README.md、学習記録）で管理
- プロジェクト間の比較が容易
